## Key Learnings & Gotchas

- **Issue comments after filing**: Both #6 and #11 had significant additional context added post-filing (iatric-mfg follow-ups). Always re-read issue comments before closing — the extra scope (minimal-transformation principle, REMOTE_*/PREPROD_* split caveat, dry-run /tmp leak) doubled the value of those PRs.
- **#10 closed prematurely**: Closed #10 before reading the follow-up comment that extended classname_value_remapping to UserForms fields + anti-pattern. Reopened + re-closed after incorporating the extra scope into PR #23. Check all comments before closing.
- **Block.Published is always 0**: SS3 `Block.Published` column looks like a live/draft flag but is a versioning artifact — always 0. The real live gate is a row in `Block_Live`. This was causing draft-only blocks to migrate and render on the SS4 frontend.
- **MigrateContentToElement is harmful on native-$Content themes**: Running it moves `SiteTree.Content` into Elemental and blanks the field, breaking blog excerpts + page layout — for zero benefit if the theme renders `$Content` natively. Documented in issue #18 (still open).
- **REMOTE_*/PREPROD_* misconfiguration**: When deploy.sh reads REMOTE_* instead of PREPROD_*, sync and deploy silently target the same server. No error output. The team spent significant time QA-ing changes against what turned out to be the wrong server.
- **pr-agent on doc PRs**: Checks ticket compliance strictly. Missed items: PHPCS path needed `app/tests/`, explicit `composer require` command, and the cross-reference link. Good to run `improve` pass too for large doc PRs.
