## Key Learnings & Gotchas

- **pr-agent "partially compliant" false negatives are common on docs PRs.** The #7 flag ("master migration step not covered") was refuted inline. Always read the non-compliant detail before acting; the issue docs often already cover the point in different words.
- **Both skill repos draft PRs by default** — `gh pr ready <n>` needed before merge. (Both #39 and #7 were in draft state.)
- **#33 issue forward-reference was a dead link to a GitHub issue URL.** The Phase 1 `core-tools` removals row was already well-written and needed only the link target swapped to the new reference file path.
- **#35 affects both `-legacy` and upgrade DDEV instances.** The Mutagen `upload_dirs` fix must be applied to both `.ddev/config.yaml` files — the legacy instance hits the identical conflict on its first prod sync.
- **#36 symptom is easy to misread.** A 3-second whole-run with all-ERROR isn't a broken capture script — it's the hostname never resolving. The 127.0.0.1 ports fix it, but ports are dynamic (reset after ddev restart/poweroff).
- **`silverstripe-skills` branch `feature/ss3-ss4-block-migration-patterns` and `docs/vr-same-machine-22` are still present on the remote** (old unmerged work from before this session). Not touched here — they predate this session's issue set.
