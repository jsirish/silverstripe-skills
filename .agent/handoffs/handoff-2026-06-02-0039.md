## Objective

## Key Learnings & Gotchas

- **BSD `sed -i ''` in a shell loop fails silently** when the file list comes from command substitution — the filenames get passed as part of the sed expression, not as separate arguments. Switch to `perl -i -pe` for in-place multi-pattern replacements on macOS.
- **`git reset` after `git rm --cached` reverts the untrack** if you soft-reset commits that include the untrack. Must redo the `git rm --cached` after the reset.
- **Allowlist `.gitignore` + `git rm --cached` interaction:** After removing `!/.agent/` from the allowlist, `git add .agent` is rejected as ignored — the untrack must be staged via `git rm --cached` (which bypasses `.gitignore`), not `git add`.
- **Sensitivity assessment findings:** No credentials/secrets anywhere. Client project names (`rockline-iatric`, `youth-sailing`, `sheboygan-safeharbor`) were already in 32 tracked skill-doc locations before the `.agent/` handoffs added more. History purge would be needed to fully eliminate them from the public record.
- **`google-ads-manager`** is still in the `dynamic` org and is private — not affected by this cleanup.
