## Key Learnings & Gotchas
- `gh api -X POST repos/dynamic/<repo>/transfer` returns the old `full_name` immediately (transfer is async on GitHub's side); need a ~4s sleep before `gh repo view jsirish/<repo>` resolves
- `git add skills/*/metadata.json` doesn't expand the glob when run via `git -C <path>` from a different cwd — must `cd` into the repo first
- Both repos' local git identity was the `Builder <builder@example.com>` placeholder — set per-repo to `Jason Irish <jsirish@users.noreply.github.com>` for these commits
- `silverstripe-skills` remote was already SSH; `workflow-skills` was HTTPS — standardised both to SSH
- `google-ads-manager` deliberately left under `dynamic` — has in-flight remote work, not in scope
- Historical session logs in `.agent/handoffs/` reference `dynamic/` URLs — left as-is (point-in-time records, not live config)
