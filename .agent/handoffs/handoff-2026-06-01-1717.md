## Key Learnings & Gotchas

**skills.sh format requirements:**
- `name:` in `SKILL.md` frontmatter must exactly match the directory slug (case-sensitive, kebab-case)
- Each skill needs `skills/<slug>/metadata.json` with `{ "author": "...", "version": "1.0.0" }`
- `skills.sh.json` needs `$schema` and `notGrouped` fields
- `npx skills add -a claude-code -g` installs to `~/.claude/skills/` (real dirs); does NOT update `~/.agents/skills/`

**~/.agents/skills/ vs ~/.claude/skills/:**
- `~/.agents/skills/` = skills.sh global store for all agents (Antigravity, Cline, etc.) — also where pre-npx manual skills lived
- `~/.claude/skills/` = Claude Code-specific install target (populated by `-a claude-code`)
- After `npx skills add`, the old `~/.claude/skills/` symlinks pointed at `~/.agents/skills/` and were replaced by real dirs; the old `~/.agents/skills/` orphans had to be manually removed

**visual-regression-upgrade review findings fixed:**
1. Credential scope: shared `--auth` was sent to both prod and UAT → added `--prod-auth`/`--local-auth` per-env flags
2. Masking: inline styles didn't cover `img`/`iframe`/`canvas` → replaced with absolute overlay divs via `getBoundingClientRect`
3. Crawler redirect scope: `requests.get()` didn't check if redirects stayed on-host → added `_hosts_match()` check
4. Path parsing: `--paths` comma split breaks on paths with commas → added `--paths-file` for newline files
5. Mask query strings: `/?flush=1` didn't match `/` mask → strip query before lookup
6. `--initScript` false reference in silverstripe.md → corrected to describe actual limitation
7. Crawler content-type case-sensitivity: `"text/html"` check missed `Text/HTML` → added `.lower()`

**One-sided element masking is inherently limited:** Masks create overlays where elements exist. For UAT-only elements (dev bars), prod has no overlay → residual diff. Only mitigation is disabling the module or accepting noise.
